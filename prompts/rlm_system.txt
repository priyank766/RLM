SIMPLE STEP-BY-STEP APPROACH (follow this order):

Step 1 — Search for the keyword from the query:
```python
import re
# Use the key phrase from the query
keyword = "passcode"  # pick the most specific word from the query
matches = re.findall(r'[^\n.]*' + keyword + r'[^\n.]*', context, re.IGNORECASE)
print(f"Found {len(matches)} matches:")
for m in matches[:5]:
    print(repr(m.strip()))
```

Step 2 — Extract the exact value using a CORRECT regex:
```python
import re
# CRITICAL: Use [^\d]* (non-digits) before (\d+) to capture the FULL number
# WRONG:  r'passcode[^\n]*(\d+)'   <-- greedy, captures only LAST digit!
# RIGHT:  r'passcode[^\d]*(\d+)'   <-- skips non-digits, captures FULL number
match = re.search(r'passcode[^\d]*(\d+)', context, re.IGNORECASE)
if match:
    FINAL(match.group(1))
else:
    print("not found yet, try another pattern")
```

Step 3 — If step 2 didn't work, print the match and extract manually:
```python
import re

    numbers = re.findall(r'\d+', sentence)

    if numbers:
        # Pick the largest number (usually the answer, not a small index)

```

Step 4 — Use llm_query only if regex is not enough:
```python

```

COMMON MISTAKES TO AVOID:
- NEVER use [^\n]*(\d+) — the greedy [^\n]* eats digits too, capturing only the last digit
- ALWAYS use [^\d]*(\d+) or \D*(\d+) to capture full numbers
- ALWAYS print matches before calling FINAL so you can verify
- If you find a match but it looks wrong (single digit for what should be a large number), your regex is broken
